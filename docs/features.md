# 実装済み機能一覧

## タイマー種別

### 1. カウントダウンタイマー (`countdown`)

目標日時までの残り時間を表示するタイマー。

**フィールド:**
- `name`: タイマー名
- `targetDate`: 目標日時（ISO 8601形式、UTC）
- `tags`: タグリスト（オプション）

**表示:**
- 残り時間: 「X日 Y時間 Z分」形式
- 進捗率: 作成時刻から目標日時までの経過割合
- 円形プログレスバー: 緑→青→黄→オレンジ→赤（80%→20%）
- 期限切れ後: 「期限切れ」と表示、赤色強調

**計算ロジ:**
- 進捗率 = (現在時刻 - 作成時刻) / (目標日時 - 作成時刻) × 100
- 残り時間 = 目標日時 - 現在時刻

**ユースケース:**
- プロジェクト納期管理
- イベントまでのカウントダウン
- 試験日までの残り日数

---

### 2. 経過時間タイマー (`elapsed`)

開始日時からの経過時間を表示するタイマー。

**フィールド:**
- `name`: タイマー名
- `startDate`: 開始日時（ISO 8601形式、UTC）
- `tags`: タグリスト（オプション）

**表示:**
- 経過時間: 「X日 Y時間 Z分」形式
- 進捗率: なし
- サブテキスト: 「SINCE {開始日時}」

**計算ロジック:**
- 経過時間 = 現在時刻 - 開始日時

**ユースケース:**
- プロジェクト開始からの経過日数
- 記念日からの日数
- 習慣継続日数

---

### 3. カウントダウン→経過タイマー (`countdown-elapsed`)

期限前はカウントダウン、期限後は経過時間を表示するタイマー。

**フィールド:**
- `name`: タイマー名
- `targetDate`: 目標日時（ISO 8601形式、UTC）
- `tags`: タグリスト（オプション）

**表示:**
- 期限前: カウントダウンタイマーと同じ
- 期限後: 経過時間「X日 Y時間 Z分」形式
- サブテキスト: 期限前「UNTIL {目標日時}」、期限後「SINCE {目標日時}」

**計算ロジック:**
- 現在時刻 < 目標日時: カウントダウンモード
- 現在時刻 >= 目標日時: 経過時間モード

**ユースケース:**
- 期限の前後で表示を切り替えたい場合
- 締切までと締切後の両方を追跡したい場合

---

### 4. スタミナ回復タイマー (`stamina`)

ゲームのスタミナのように、時間経過で値が回復するタイマー。

**フィールド:**
- `name`: タイマー名
- `currentValue`: 現在値
- `maxValue`: 最大値
- `recoveryIntervalMinutes`: 回復間隔（分）※非推奨
- `recoveryIntervalSeconds`: 回復間隔（秒）※推奨
- `lastUpdatedAt`: 最終更新日時（ISO 8601形式、UTC）
- `tags`: タグリスト（オプション）

**表示:**
- 現在値/最大値: 「50/100」形式
- 進捗率: 現在値/最大値 × 100
- 次回回復まで: 「5分12秒後に回復」
- 全回復時刻: 「12/31 23:59 全回復」（絶対時刻）
- 円形プログレスバー: 緑→青→黄→オレンジ（進捗に応じて）

**計算ロジック:**
- 経過時間 = 現在時刻 - 最終更新時刻
- 回復量 = floor(経過時間 / 回復間隔)
- 新しい現在値 = min(現在値 + 回復量, 最大値)
- 次回回復まで = 回復間隔 - (経過時間 % 回復間隔)
- 全回復時刻 = 現在時刻 + (最大値 - 新しい現在値) × 回復間隔

**ユースケース:**
- ゲームのスタミナ管理（例: アークナイツ・エンドフィールド 7分12秒ごとに1回復）
- リソースの自動回復追跡
- タスク実行回数の管理

---

### 5. 定期増加タイマー (`periodic-increment`)

特定の時刻に値が増加するタイマー。

**フィールド:**
- `name`: タイマー名
- `currentValue`: 現在値
- `maxValue`: 最大値
- `incrementAmount`: 増加量
- `scheduleTimes`: 増加時刻リスト（「HH:MM」形式）
- `lastUpdatedAt`: 最終更新日時（ISO 8601形式、UTC）
- `tags`: タグリスト（オプション）

**表示:**
- 現在値/最大値: 「30/100」形式
- 進捗率: 現在値/最大値 × 100
- 次回増加まで: 「6時間12分後に+10」
- 次回増加時刻: 「12/31 18:00 +10」
- 円形プログレスバー: 緑→青→黄→オレンジ（進捗に応じて）

**計算ロジック:**
- スケジュール時刻が過ぎている場合、増加量を加算
- 新しい現在値 = min(現在値 + (過ぎたスケジュール数 × 増加量), 最大値)
- 次回増加時刻 = 最も近い未来のスケジュール時刻

**ユースケース:**
- ゲームのログインボーナス管理
- 定時タスクのリマインダー
- 定期イベントの追跡

## UI/UX機能

### ビュー切り替え

**カードビュー:**
- 大きな円形プログレス（128×128px）
- ゆったりしたレイアウト
- グリッド表示（PC: 3列、タブレット: 2列、モバイル: 1列）

**リストビュー:**
- コンパクトな円形プログレス（48×48px）
- 1行にまとめた表示
- 縦スクロールで多くのタイマーを一覧

**実装:**
- Alpine.js による状態管理
- LocalStorage でビュー設定を永続化
- アイコンボタンで切り替え

---

### タグ機能

**作成・編集:**
- カンマ区切りで複数タグ入力（例: `仕事, 重要, プロジェクトA`）
- 前後の空白は自動トリミング
- 空タグは除外

**表示:**
- 青いバッジとして表示
- ダークモード対応（`bg-blue-900/30 text-blue-300`）
- タイマーカード・リストアイテムの両方に表示

**データ構造:**
- DB: JSON文字列として保存（`'["work","important"]'`）
- アプリケーション: `string[]` 配列

---

### フィルター機能

**種別フィルター:**
- ドロップダウンで選択
- オプション: すべて / カウントダウン / 経過時間 / カウントダウン→経過 / スタミナ回復 / 定期増加

**タグフィルター:**
- トグルボタンで複数選択可能
- 選択中は青色（`bg-blue-600`）、未選択は白色（`bg-white`）
- OR条件: いずれかのタグに一致すれば表示

**クリアボタン:**
- 全フィルターを一括解除

**実装:**
- Alpine.js によるクライアント側フィルタリング
- `x-show` で表示/非表示を制御（ページリロード不要）
- リアルタイムで反映

---

### ダークモード

**切り替え:**
- 2つのボタン（Light / Dark）
- 選択中は青色で強調
- LocalStorage で設定を永続化

**対応範囲:**
- 全ページ・全コンポーネント
- 背景色、テキスト色、ボーダー色
- プログレスバー、ボタン、モーダル

**実装:**
- Alpine.js によるリアクティブな切り替え
- Tailwind CSS の `dark:` クラスを使用
- `prefers-color-scheme` メディアクエリでデフォルト設定

---

### レスポンシブデザイン

**ブレークポイント:**
- モバイル（< 768px）: 1列、縦スクロール
- タブレット（768px - 1024px）: 2列グリッド
- デスクトップ（> 1024px）: 3列グリッド

**レイアウト:**
- PC: 画面全体を活用（`max-w-7xl`）
- モバイル: 中央寄せ、パディング調整

**タッチ対応:**
- ボタンサイズ: 最低44×44px（iOS Human Interface Guidelines）
- タップ領域の確保

---

## データ管理機能

### リアルタイム更新

**更新頻度:**
- 1秒ごとに自動更新（`setInterval(1000)`）

**更新内容:**
- 残り時間/経過時間
- 進捗率
- 次回回復/増加時刻
- スタミナ現在値

**実装:**
- Alpine.js の `timerDisplay()` data
- 各タイマーカードで独立して更新
- DOM 操作は Alpine.js のリアクティブシステムに委譲

---

### タイムゾーン対応

**入力:**
- `<input type="datetime-local">` で JST として入力
- `datetimeLocalToISO()` で UTC に変換して保存

**表示:**
- サーバーから UTC で受け取る
- `isoToDatetimeLocal()` で JST に変換して表示

**計算:**
- サーバー側（Cloudflare Workers）は UTC で計算
- クライアント側は Date オブジェクトで自動的にローカルタイムゾーンを使用

**実装:**
- `src/lib/timezone.ts` で変換関数を提供
- ISO 8601 形式で統一

---

### CRUD操作

**作成（Create）:**
- `POST /api/timers`
- Zod バリデーション
- DB に保存後、タイマー一覧にリダイレクト

**読み取り（Read）:**
- `GET /`: 全タイマー取得
- `GET /timers/:id/edit`: 特定タイマー取得（編集用）

**更新（Update）:**
- `POST /api/timers/:id` with `_method=put`
- タイマー種別は変更不可（データ整合性のため）
- Zod バリデーション
- DB 更新後、タイマー一覧にリダイレクト

**削除（Delete):**
- `DELETE /api/timers/:id`
- htmx による部分更新（ページリロード不要）
- 削除確認モーダル表示

---

## セキュリティ機能

### XSS対策

- Hono JSX による自動エスケープ
- JSON シリアライズ時の `<` → `\u003c` 変換
- `x-text` によるテキストバインディング（innerHTML 不使用）

### SQLインジェクション対策

- Drizzle ORM によるプリペアドステートメント
- 生SQL不使用

### CSRF対策

- Same-Site Cookie（Cloudflare Pages デフォルト）
- 重要操作は POST/PUT/DELETE メソッド

---

## パフォーマンス機能

### SSR + クライアント側ハイドレーション

- 初回ロードは SSR で高速表示
- Alpine.js でクライアント側リアクティビティ追加

### CDN エッジキャッシング

- 静的アセットは Cloudflare CDN でキャッシュ
- HTML はエッジで生成（低レイテンシ）

### クライアント側フィルタリング

- フィルター操作はクライアント側で実行
- サーバーリクエスト不要
